# API Map

Base route convention: `api/[controller]`.

Authentication: JWT Bearer. Most controllers require `[Authorize]`; selected actions require roles.

## Controllers and Endpoints

### Auth

Controller: `AssetManagement.API/Controllers/AuthController.cs`

| Method | Route | Auth | Service |
|---|---|---|---|
| `POST` | `/api/auth/login` | anonymous | `IAuthService.LoginAsync` |
| `POST` | `/api/auth/logout` | authenticated | `IAuthService.LogoutAsync` |
| `GET` | `/api/auth/me` | authenticated | `IAuthService.GetCurrentUserAsync` |

### Categories

Controller: `AssetManagement.API/Controllers/CategoriesController.cs`

Pattern mirrors assets: list, paged, get by id, create/update/delete/restore. Create/update require Admin or Manager; delete/restore require Admin.

### Locations

Controller: `AssetManagement.API/Controllers/LocationsController.cs`

Pattern mirrors categories. Create/update require Admin or Manager; delete/restore require Admin.

### Users

Controller: `AssetManagement.API/Controllers/UsersController.cs`

Provides list, paged, get by id, create, update, delete, activate, deactivate, restore. Administrative operations are role protected, mostly Admin.

### Audit Logs

Controller: `AssetManagement.API/Controllers/AuditLogsController.cs`

Admin-only. Provides paged logs and filters by entity/user.

### Data Retention

Controller: `AssetManagement.API/Controllers/DataRetentionController.cs`

Admin-only. Provides cleanup execution and retention status through `IDataCleanupService`.

## Response Pattern

Controllers usually receive `Result` or `Result<T>` from services:

- success with data: `Ok(result.Data)`
- create success: `CreatedAtAction(...)`
- delete success: `NoContent()`
- failure: mostly `BadRequest(result.Error)`, with some `NotFound(result.Error)`

Exception middleware handles thrown `NotFoundException`, `ValidationException`, `BusinessException`, and `UnauthorizedAccessException`, but normal business failures are generally returned as `Result`.

## Authorization Model

Backend roles are enum values:

- `Admin`
- `Manager`
- `Employee`

JWT tokens are generated by `JwtTokenGenerator` and validated by infrastructure auth configuration.

Common policy by convention:

- Any authenticated user can read most asset/category/location/assignment lists.
- Admin and Manager can create/update operational records.
- Admin can delete/restore and access audit/retention endpoints.

## API/Frontend Alignment

Frontend API services live in `../../Frontend/src/app/core/services`:

- `auth.service.ts` -> `/auth`
- `asset.service.ts` -> `/assets`
- `assignment.service.ts` -> `/assignments`
- `category.service.ts` -> `/categories`
- `location.service.ts` -> `/locations`
- `user.service.ts` -> `/users`
- `audit-log.service.ts` -> `/auditlogs`

When changing an endpoint, update both the controller and matching Angular service.
